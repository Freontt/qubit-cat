## О проекте

### Проблема
В современном цифровом мире приватность общения находится под угрозой. Централизованные мессенджеры собирают метаданные, хранят историю переписки на своих серверах и подвержены утечкам данных. Пользователи не могут быть уверены в конфиденциальности своих личных, деловых или семейных бесед.

### Решение
QubitCat предлагает принципиально новый подход к приватному общению:
- **End-to-end шифрование на уровне протокола с использованием современных криптографических стандартов**
- **Децентрализованное хранение данных через распределённую сеть узлов**
- **Открытый API для создания пользовательских плагинов и расширений**
- **Самоуничтожающиеся сообщения с настраиваемым временем жизни**
- **Отсутствие метаданных — мы не храним информацию о том, кто с кем общается**

### Целевая аудитория
- **Государственные деятели и дипломаты для защищённых переговоров**
- **Независимые журналисты и СМИ для конфиденциального общения с источниками**
- **Бизнесмены и корпорации для коммерческой тайны**
- **Активисты и правозащитники в странах с цензурой**
- **Обычные пользователи, ценящие приватность в личной переписке**

### Основная цель
Создать эталонный open-source мессенджер, который объединяет максимальную безопасность с удобством использования. Проект развивается сообществом для сообщества, обеспечивая прозрачность и независимость от корпоративных интересов.

### Философия
1. **Приватность по умолчанию** — все функции проектируются с приоритетом безопасности
2. **Прозрачность** — весь код открыт для аудита
3. **Децентрализация** — никаких центральных серверов, которые могут стать точкой отказа
4. **Доступность** — безопасность должна быть доступна каждому, а не только экспертам

## Возможности

### Безопасность
- **End-to-end шифрование**
- **Протокол двойного криптографического затвора**
- **Проверка ключей**
- **Защита от MITM-атак**
- **Самоуничтожающиеся сообщения**

### Децентрализация
- **P2P-сеть**
- **Распределённое хранение**
- **Нодовая инфраструктура**
- **Отказоустойчивость**

### Расширяемость
- **Плагиновая система**
- **Пользовательские темы и интерфейсы**
- **Интеграции с внешними сервисами через плагины**
- **Кастомные протоколы шифрования для продвинутых пользователей**

### Общение
- **Групповые чаты до 100 участников с групповым шифрованием**
- **Файлообмен с шифрованием**
- **Голосовые сообщения**
- **Видеозвонки**

### Анонимность
- **Регистрация без номера телефона и email**
- **Генерация уникального ID на устройстве**
- **Защита от анализа трафика**

## Технологии

### Основной стек
- **Клиент**: DearImGUI
- **P2P**: WebRTC, libp2p
- **Криптография**: Libsodium, Crypto++
- **Базы данных**: SQLite (локально) и PostgreSQL (Сервер)

### Инфраструктура
- **TURN-серверы**
- **DHT-сеть**
- **Пользовательские устройства**
- **Центральный сервер**

### Безопасность
- **Аудит кода**
- **Тестирование**
- **Обновление зависимостей**
